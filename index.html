Connect GitHub & Webstorm

  1. (Ctrl+Alt+S) in Webstorm for system preferences
  2. Select Git under version control, and enter the path to the git.exe
  3. Still in system preferences, add you GitHub password to Webstorm by selecting Appearance and Behavior > System Settings> Passwords

Create a Repository in GitHub

  1. On GitHub, click the "+" sign in the upper right corner
  2. Choose "Create New Repository"
  3. Be sure to make the repository public and add the readme file
  4. Click "create"

Create a Repository from Webstorm

In Webstorm, select VCS and Import into Version Control, then follow the steps below to import a repository from GitHub

  1. From the Main page, select Checkout from version control > Git OR From within Webstorm, select VCS > Checkout from version control > Git
  2. Enter GitHub repository name
  3. Enter local path name

Creating a Webstorm File

To create a Webstorm file simply click "File" > HTML > HTML5 or file > Stylesheet

Adding files to GIT

  1. Open Add to GIT dialog
  2. CLick "add" which adds to the local file system
  3. Commit your changes by clicking "Commit"
  4. Push change to remote repository by clicking "Ctrl > Shift > K" OR "VCS > Git > Push"

Congratulations! Your file is now on GitHub

Set up GitHub Pages

  1. In GitHub, click "Settings"
  2. Check the repository name
  3. Select "Master branch"
  4. Note the published URL
  5. Finally, check your GitHub page by copying thr GitHub.io URL into a browser

Glossary of Key Terms

  • Branch - In Git, a powerful mechanism that allows you to diverge from the main development line
  • Clone -
  • Commit - way to share your work after you have added new files or modified existing files by locally recording a snapshot of your repository
  • Fetch - method of taking changes from the upstream and allowing all new data from commits that were made since you last synced with the remote repository to be downloaded into your local copy
  • GIT - a software for tracking changes in any set of files
  • Github - a provider of Internet hosting for software development and version control using Git
  • Merge - a way of updating a working copy if push is rejected by combining with a second copy of the branch
  • Merge Conflict - If there are unresolved conflicts left after a merge, this node will appear in the corresponding changelist in the Local Changes view with a link to resolve them
  • Push - a term that refers to the method of moving your commits to a remote repository, so they can be available to others
  • Pull - used if you need to get changes into the current branch from another branch instead of its remote tracked branch
  • Remote - a Git repository that's hosted on the Internet or another network
  • Repository - contain a collection of files of various different versions of a Project
